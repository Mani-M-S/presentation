Boucle agentique + RAG — Le LLM décide lui-même

Pourquoi une boucle agentique ?
  • Le LLM décide lui-même quels outils appeler et dans quel ordre
  • Si ChromaDB ne trouve rien → scraping live du site Netethic automatiquement
  • Pas de pipeline rigide à maintenir — s'adapte aux nouvelles questions
Patron Strategy — Multi-LLM sans friction
AGENT_PROVIDER=claude → Anthropic API AGENT_PROVIDER=gemini → Google AI API Interface commune : run_agent(session, msg, lang) Changement de LLM sans modifier le code
RAG public — Ancrer dans les faits
INDEXATION
Site Netethic → chunks → sentence-transformers (384d) → ChromaDB
REQUÊTE
"combien ça coûte" → top-3 chunks "tarifs" → injectés dans le prompt
Sans RAG = hallucination
Le LLM ne connaît pas les tarifs spécifiques de Netethic → il invente des réponses plausibles mais fausses.
