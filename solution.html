Choix Technologiques — Backend & Infrastructure

Chaque technologie est justifiée

F
API Layer
FastAPI
  • Framework Python async — support natif des requêtes concurrentes
  • Validation automatique via Pydantic (OWASP LLM07)
  • Documentation OpenAPI générée automatiquement
  • Déployé dans chaque conteneur Docker du module
vs Flask : sans async natif. Django : trop lourd pour un service IA dédié.
R
Mémoire conversationnelle
Redis
  • Stockage en mémoire ultra-rapide — latence < 1 ms
  • Historique de conversation par session (6 derniers échanges)
  • TTL automatique — suppression après 24h (RGPD)
  • Cache sémantique des réponses PDF (TTL 48h)
vs PostgreSQL : conversations éphémères — Redis offre le TTL natif et le type liste idéal.
D
Orchestration
Docker Compose
  • Chaque module dans son conteneur isolé
  • Déploiement reproductible en une commande : docker compose up
  • Réseau interne privé — services non exposés à l'extérieur
  • Volumes persistants pour ChromaDB
vs Kubernetes : complexité injustifiée pour 4 services. Compose suffit pour le contexte SaaS actuel.
P
Données métier
PostgreSQL
  • Stockage des RDV, signalements et bugs (Module 2)
  • Données persistantes et transactionnelles — ACID complet
  • Pydantic valide les données avant chaque insertion
  • Séparation claire : état conversatoire (Redis) vs données métier (PG)
vs SQLite : pas adapté à un SaaS multi-utilisateurs. MongoDB : schéma strict requis pour les RDV.

Choix Technologiques — IA & LLM Stack

Le cerveau du système

V
Moteur d'inférence LLM
vLLM
  • Serveur d'inférence pour Mistral-7B en local (kaisens.fr)
  • PagedAttention — optimisation mémoire GPU
  • API compatible OpenAI — zéro friction d'intégration
  • Batching continu — débit élevé en production
vs Ollama : conçu pour l'usage personnel — vLLM offre le batching production.
C
Base vectorielle
ChromaDB
  • Stockage et recherche par similarité cosinus des embeddings
  • Filtrage par métadonnées — scope par catégorie (Module 2)
  • RAG juridique (Module 4) — Code Pénal indexé
  • Persistance disque — survie aux redémarrages
vs Pinecone : cloud uniquement — données sensibles restent en local.
LLM Externes — Module 1
Claude / Gemini
  • Patron Strategy — bascule via variable d'env AGENT_PROVIDER
  • Tool use natif — boucle agentique pour Noa
  • Aucun remplacement de code requis pour changer de LLM
Embeddings locaux
sentence-transformers
  • multilingual-e5-base — 384d, multilingue
  • Aucun appel réseau, aucun coût d'API pour les embeddings
  • Comprend le sens — "combien ça coûte" → chunk "tarifs"
